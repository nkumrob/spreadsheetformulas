## 2026-07-08 — MVP skeleton built (milestones M1 + most of M2)
- Stack in place: Next.js 14.2 (App Router, TS strict) + Tailwind 3.4 + Vitest + Zod. All 32 routes prerender statically; first-load JS < 100 kB.
- **Design system: "The Modern Ledger"** — user demanded expert-level, non-AI-slop design. Direction: warm paper (#FAF7F1), ink (#1C1A16), ledger green (#166B4A), rust for errors (#B3402E), hairline rules, faint cell-grid backgrounds. Fonts: Instrument Serif (display), Schibsted Grotesk (UI), Spline Sans Mono (formulas) via next/font. Signature elements: FormulaBar (fx glyph + copy) and SpreadsheetTable (real column letters/row numbers, tinted result column). Keep ALL future UI consistent with this — no Inter, no purple gradients.
- Content is typed TS data (content/*.ts) validated by Zod at build time — NOT MDX (deviation from SF-010; structured fields fit the §10 page anatomy better). 12 formula pages + 4 error-fix pages seeded, all formulas verified logically; error fixes live at /errors/[slug], formulas at /formulas/[slug] (lib/paths.ts routes by kind).
- Search is build-time index + client-side scoring (lib/search.ts), tested against PRD §11 queries. Platform preference persists in localStorage (sf-platform). Feedback + copy + search events funnel through lib/analytics.ts (Plausible-ready, no-op until script added).
- Newsletter posts to /api/subscribe; forwards to NEWSLETTER_WEBHOOK_URL env var when set.
- **Port 3000 is occupied by another app on this machine ("NATO Phonetic")** — always use another port (e.g. 4123) for local servers.
- Visual QA done via Playwright screenshots at 1440px and 375px on home, formula, error, search, categories — all clean, no horizontal scroll on mobile.
- Remaining for launch (see mvp-execution-tickets.md): 38 more formula pages, 6 more error pages, templates (SF-041/042/087), explainer/fixer tools (E5), real analytics + newsletter provider wiring, e2e suite (SF-090).

## 2026-07-08 — MVP content-complete + tools + templates + competitive research
- Content: 56 pages total — 46 formula pages + 10 error-fix pages (all PRD §17 launch topics). 34 formula + 6 error pages were authored by 6 parallel subagents against docs/content-authoring-guide.md; all validated by the Zod build gate + 34 unit tests. Formula spot-checks passed (agents produced correct formulas AND arithmetically-correct sample data).
- Tools (E5, no AI/API needed): lib/formula-check.ts = deterministic checker (smart quotes, unbalanced parens incl. quoted-string awareness, VLOOKUP 3-arg detection w/ auto ",FALSE" correction, locale semicolons, ~110-function catalog for #NAME? detection, error-value → /errors/ links). lib/formula-explain.ts = rule-based explainer with curated function guide + top-level arg splitting. Both at /tools/fix-formula and /tools/explain-formula; /tools/ai is the AI-generator waitlist.
- Templates: 5 real .xlsx with working formulas generated by scripts/build_templates.py (openpyxl, installed --user; dates MUST be datetime.date not strings or formulas break). Pages at /templates/[slug] with mini-spreadsheet previews + download tracking.
- Search ranking lesson: with many pages sharing a function, dedicated-tutorial ranking needs (a) summed field weights not max, (b) primary-function boost, (c) seoTitle-starts-with-term boost. See lib/search.ts.
- e2e: scripts/e2e_smoke.py (14 checks — copy button, fixer correction, explainer, downloads, 404, mobile overflow) run via webapp-testing skill's with_server.py on port 4123. Mobile overflow fix pattern: grid/flex items need min-w-0 when children use whitespace-nowrap.
- Competitive research: docs/research-formula-bots.md (July 2026). Key takeaways: Formula Bot pivoted upmarket ($18-149/mo, vacating simple formula help); GPTExcel is the $9 textbox generator; Arcwise abandoned the niche; MS Copilot's own docs disclaim formula accuracy; Exceljet lost ~50% traffic to an AI content clone (text-only content is not a moat). Our moat = visible testing evidence + deterministic pre-checks + sample-data-with-every-answer + Excel-vs-Sheets axis + templates for retention. P0 recs largely already built into the product.
- Still open before real launch: hosting/domain, Plausible/PostHog script + NEWSLETTER_WEBHOOK_URL env, Google Sheets copy-links for templates, remaining §17 authoring polish (4 launch-list topics folded into error pages), Search Console submission.
