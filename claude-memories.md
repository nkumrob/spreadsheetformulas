## 2026-07-08 — Layer-1 formula verification harness (engine execution)
- Every content page now carries an executable `verification` block (lib/schema.ts): raw grids + pinned expected cell values, executed by HyperFormula 3.3 in tests/formula-execution.test.ts with vi.setSystemTime frozen to 2026-07-08. Coverage: 52/56 pages engine-verified; 4 legitimately null (UNIQUE/RANK/TEXTSPLIT engine gaps). Coverage tests make silent gaps impossible: formula-kind pages MUST have a block or use an engine-missing function.
- HyperFormula quirks learned: (1) bare TRUE/FALSE literals are NOT parsed — harness aliases them via namedExpressions [{name:"FALSE",expression:"=FALSE()"}]; symptom is #NAME? swallowed by IFERROR into wrong-looking values. (2) Whole-column refs (A:A) cause #CYCLE! if the formula cell sits in a referenced column — use bounded ranges or another column. (3) 6-arg XLOOKUP with search mode -1 IS supported. (4) Engine-missing: LOOKUP, AVERAGEIFS, TEXTBEFORE/AFTER/TEXTSPLIT, UNIQUE, SORT, RANK, REGEXEXTRACT, QUERY.
- Verification blocks also test commonMistakes claims (naive formula misbehaves AND guarded formula works) — e.g. fix-div0 pins both A2/B2→"#DIV/0!" and the IF guard→"".
- Product surface: formula pages show a green "✓ Engine-verified against the sample data above" badge (FormulaArticle reads formula.verification); /how-we-test explains the methodology with live coverage counts; linked from footer + homepage trust strip.
- tsconfig needed explicit "target": "ES2017" once test files entered the build's type graph (matchAll iteration).
- One verification agent died on a session limit mid-file (errors3.ts) — its errors2.ts work was complete and valid; I finished errors3.ts by hand. Lesson: agent batches should be small enough to survive limits; always check git diff after a dead agent.
- Remaining verification layers (designed, not built): L2 dual-engine cross-check (LibreOffice headless recalc of templates, Google Sheets API ground truth), L4 corpus/property-based testing of the fixer (Enron/EUSES corpora, fast-check).
